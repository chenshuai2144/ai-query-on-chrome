此问题是由于 React 官方在 Strict Mode 下出于性能和稳定性考虑，不建议使用 `findDOMNode` API，并且可能在未来的版本中完全移除该 API。

为了避免这个问题，可以尝试使用 React 的 `ref` API 替代 `findDOMNode`。具体包括：

1. 在组件中使用 `React.createRef()` 创建一个 `ref` 对象
2. 在渲染的元素上添加 `ref={this.xxxRef}` 属性
3. 在需要使用 DOM 元素时，通过 `this.xxxRef.current` 获取 DOM 元素

示例代码如下：

```jsx
import React from "react";

class MyComponent extends React.Component {
  constructor(props) {
    super(props);
    this.myRef = React.createRef();
  }

  componentDidMount() {
    const domNode = this.myRef.current; // 获取 DOM 元素
    // 做一些关于 DOM 元素的操作
  }

  render() {
    return <div ref={this.myRef}>Hello, World!</div>;
  }
}
```

另外，Ant Design 官方也提供了一个替代方法。您可以使用 `ReactDOM.findDOMNode()` 替代 `findDOMNode`，并将其传递到组件的 `ref` 属性中。示例代码如下：

```jsx
import React from "react";
import ReactDOM from "react-dom";
import { Form, Input, Button } from "antd";

const MyForm = () => {
  const [form] = Form.useForm();
  const inputRef = React.useRef(null);

  React.useEffect(() => {
    const inputNode = ReactDOM.findDOMNode(inputRef.current); // 获取 DOM 元素
    // 做一些关于 DOM 元素的操作
  }, []);

  return (
    <Form form={form}>
      <Form.Item name="username" label="Username">
        <Input ref={inputRef} />
      </Form.Item>
      <Form.Item>
        <Button type="primary" htmlType="submit">
          Submit
        </Button>
      </Form.Item>
    </Form>
  );
};
```
