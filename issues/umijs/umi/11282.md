# umi3.x如何在登录后根据用户配置动态修改路由？

`type(duplicate)`

需求类似于权限路由：主要用于切换新老功能

```
{
    path: '/odd',
    component:  '/odd'
    new: {
       path: '/new',
       component: '/new'
    }
}
```

用户登录后会读取一个配置，当用开启了`new`特性就加载
` { path: '/new', component: '/new' }`
否则应该加载 `{ path: '/odd', component:  '/odd'}`

### 尝试：

虽然用access.ts配置文件也可以实现，但是`access`只能用于管是否展示，就需要写成下面这样

```
{
 path: '/odd',
 component:  '/odd'，
 access: 'notnew'
}

{
 path: '/new',
 component:  '/new'，
 access: 'new'
}
```
